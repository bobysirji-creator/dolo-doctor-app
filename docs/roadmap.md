# DO-LO Doctor App Roadmap

- [x] Stage 1 - lightweight Compose project, Doctor/Assistant login, navigation and feature skeletons
- [ ] Stage 2 - persisted local session, validation and permission-aware assistant access
- [ ] Stage 3 - doctor profile, clinic and consultation schedule management
- [ ] Stage 4 - functional appointment list and queue lifecycle with audit events
- [ ] Stage 5 - appointment availability blocks and affected-patient workflow
- [ ] Stage 6 - announcements, camps, offers and Patient App profile feed contracts
- [ ] Stage 7 - assistant creation, credentials and backend-enforced permissions
- [ ] Stage 8 - reports, feedback, multi-clinic readiness and queue-delay notices
- [ ] Stage 9 - shared backend integration with Patient App
- [ ] Stage 10 - accessibility, security, tests and release hardening

Admin App remains a separate future repository. Real providers remain disabled until the shared backend and policies are approved.
